# Reasoning Drift Study — Multi-Arm

A controlled experiment measuring how Anthropic Claude models' reasoning
quality drifts as the context window fills with adjacent-but-irrelevant
material, even at maximum extended thinking.

The task domain is **financial analysis** — a deliberately blended workload
of factual retrieval, numeric calculation, evidence-grounded reasoning, and
forward-looking thesis construction — run over Microsoft's FY2025
disclosures with adversarially-near big-tech peer 10-Ks (AAPL, GOOGL, AMZN,
META, NVDA, ORCL, CRM) as the noise corpus, because their business
complexity exercises all four reasoning modes simultaneously.

Two analyst arms are configured (Opus 4.7 and Sonnet 4.6). Only the analyst
varies between arms; methodology, prompts, rubric, materials, judge, and
extractor are held constant — both arms run at 1M context with
`thinking_effort=max`, so the comparison is apples-to-apples.

## Layout

```

├── materials/                       # corpus (MSFT 10-K, peer 10-Ks, ground truth) — shared
├── arms/
│   ├── opus-4-7/
│   │   ├── arm.lock.json            # arm config snapshot + run results
│   │   ├── data.manifest.sha256     # one line per file; integrity verifier reads this
│   │   ├── data/                    # raw, extracted, graded, manifest.sqlite, logs
│   │   └── reports/FINAL_REPORT.md  # arm-specific narrative
│   └── sonnet-4-6/                  # (not yet run)
├── cross_arm/
│   └── COMPARATIVE_REPORT.md        # generated by compare_arms.py
└── harness/
    ├── config/
    │   ├── base.yaml                # shared (extractor, judge, design grid, paths)
    │   └── arms/<arm>.yaml          # per-arm analyst override
    ├── src/                         # arm-agnostic pipeline modules
    └── scripts/                     # all take --arm <arm-name>
```

## The integrity model

This study makes a strong claim: **observed differences across arms come
from the analyst, not from the methodology, the corpus, or the measuring
instruments.** Three layered guards back that claim:

1. **`pre_registration.lock`** — single hash of `DESIGN.md + PROMPTS.md +
   RUBRIC.md` plus the materials lock hash. Edit any of those four and the
   hash changes and every arm's lock file becomes invalid.
2. **`arms/<arm>/arm.lock.json`** — pins the analyst snapshot, references
   the project pre-reg hash, snapshots the extractor + both judges (held
   constant), and records actual cost + run counts for the locked arm.
3. **`arms/<arm>/data.manifest.sha256`** — SHA-256 of every file in
   `arms/<arm>/data/` at the moment of lock.

Any of those can be re-verified offline:

```
python -m scripts.verify_arm_integrity --arm opus-4-7
python -m scripts.compare_arms                    # refuses if hashes don't match across arms
```

`compare_arms.py` reads every arm's `arm.lock.json` and **refuses to
produce a comparison** unless every arm declares the same methodology
hash, the same materials hash, the same design grid, and the same
extractor + judge configuration. That gating is what makes the cross-arm
comparison meaningful.

For the deeper rationale and the procedure for adding a new arm, see
[`ARMS.md`](ARMS.md).

## Reproducing the Opus 4.7 arm

```
git clone <repo> && cd "Opus 4.7 Reasoning Drift Study"
git checkout arm/opus-4-7/data-v1.0           # original pre-refactor state, optional
cd harness
uv sync && cp .env.example .env                # add ANTHROPIC_API_KEY
python -m scripts.verify_arm_integrity --arm opus-4-7   # confirms data byte-identical
python -m scripts.drift_analysis --arm opus-4-7         # rebuilds drift profile
```

## Running a new arm

```
# 1. Edit harness/config/arms/<arm>.yaml — set analyst snapshot + thinking effort
# 2. Verify the model accepts the chosen thinking schema
python -m scripts.smoke_test --arm <arm> --cell-fill 0.0

# 3. Pilot then full
python -m scripts.run_experiment --arm <arm> --pilot
python -m scripts.run_experiment --arm <arm> --full
python -m scripts.run_extractor --arm <arm>
python -m scripts.run_grading --arm <arm>

# 4. Lock the arm
#    Hand-write arms/<arm>/arm.lock.json mirroring opus-4-7/arm.lock.json,
#    then regenerate the data manifest and verify:
python -m scripts.verify_arm_integrity --arm <arm>

# 5. Compare
python -m scripts.compare_arms --write-report
```

## Spend and budget

Per-arm budget guards live in `harness/config/base.yaml`:

```
budget_usd:    $700   (warning threshold)
hard_stop_usd: $850   (CostTracker aborts)
```

Pricing is snapshotted into each `arm.lock.json` so cost can be honestly
reconstructed even if API rates change after an arm closes.
